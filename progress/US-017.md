# US-017: Application Entry Point and CLI Configuration

## Status: Completed

## Summary
Implemented the main entry point script with command-line argument parsing, configuration file loading, and graceful shutdown handling.

## Changes Made

### src/activity_beacon/__main__.py
- Created CLI argument parser with `--interval`, `--output`, `--debug`, and `--config` options
- Implemented `load_config()` function to read TOML configuration files from `~/.config/activity-beacon/config.toml`
- Added `merge_config()` to merge CLI args with file configuration (CLI takes precedence)
- Configured `configure_logging()` to set up daily rotating logs in `~/.logs/activity-beacon/`
- Implemented graceful shutdown handling for SIGTERM/SIGINT signals
- Created `create_capture_controller()` factory function

## Key Features
- **CLI Options**:
  - `--interval`: Capture interval in seconds (default: 30)
  - `--output`: Output directory for screenshots (default: ~/Documents/ActivityBeacon/data)
  - `--debug`: Enable debug mode with verbose console output
  - `--config`: Path to config file (default: ~/.config/activity-beacon/config.toml)

- **Configuration File**: Supports TOML format with `[capture]` and `[general]` sections
- **Signal Handling**: Graceful shutdown on SIGTERM/SIGINT
- **Logging**: Configured with daily rotation to `~/.logs/activity-beacon/`
