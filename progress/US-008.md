### Learnings for future iterations:

**Type Checking Patterns:**
- Use `collections.abc.Mapping[str, object]` instead of `dict[str, Any]` to avoid type checking errors
- Use `collections.abc.Sequence[Mapping[str, object]]` for list parameters to allow covariance
- The tuple type check `isinstance(value, tuple)` triggers type errors due to tuple element types being unknown
- json.dumps() already handles tuple serialization (converts to lists), so no manual conversion needed

**Code Style Patterns:**
- `# type: ignore[reportMissingSuperCall]` is needed for classes without explicit base class calls
- PLR6301 (method could be a function) is just a suggestion and can be ignored for private methods
- Use `from collections.abc import Mapping, Sequence` for abstract collection types

**Error Handling Pattern:**
- Store error messages in `last_error_msg` property for external access
- Log errors with context before raising
- Re-raise with wrapped message for caller convenience
